## 文档变更日志规则

**适用范围：** `docs/` 和 `doc/` 目录下所有 `.md` 文件

**规则：**
1. 每次修改上述目录中的文档，必须在文档末尾的"文档变更记录"表格中追加一行
2. 如果文档末尾尚无该表格，先创建表格（含表头）
3. 格式：

| 日期 | 变更类型 | 变更内容 | 关联提交 |
|------|---------|---------|---------|
| YYYY-MM-DD | 新增/修订/删除/废弃/重构 | 一句话摘要 | commit hash 或 - |

4. 变更类型限用上述五种，不可自定义
5. 变更内容为一句话（不超过 50 字），描述"改了什么"，不描述"为什么改"（原因看关联提交的 message）
6. 关联提交：如有对应 git commit，填 7 位短 hash；纯文档编辑无提交填 `-`
7. 同一文档一次提交中涉及多处逻辑独立的变更，每个变更点写一行
8. 仅修正拼写/排版错误（少于 5 字改动）无需记录
9. 自动生成的文档（如 API docs）不适用此规则
10. 新增行追加到表格末尾，按时间倒序排列（最新在上）—— 方便阅读时先看到最近的变更
