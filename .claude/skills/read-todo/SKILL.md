# 读取 TODO 任务列表

## 触发时机

- 用户想知道当前有哪些待办任务
- 开始新会话时快速了解工作重点
- 用户说"有什么要做"、"任务列表"、"待办事项"时

## 自动执行动作

### 1. 读取 TODO.md

使用 Read 工具读取 `docs/TODO.md`

### 2. 解析并格式化输出

提取以下信息：
- P0 高优先级任务
- P1 中优先级任务
- P2 低优先级任务
- 技术债务
- 已完成任务

### 3. 输出格式

```markdown
## 📋 PIA 待办任务

### 🔴 P0 高优先级 (影响核心体验)
| 序号 | 任务 | 状态 | 涉及文件 |
|------|------|------|----------|
| 1 | P03 精准问答快捷问题按钮 | ❌ | DetailsPage.tsx |
| 2 | P03 PDF 片段浮层集成 | ❌ | PdfFragmentOverlay.tsx |
| ... | ... | ... | ... |

### 🟡 P1 中优先级 (影响用户效率)
| 序号 | 任务 | 状态 | 涉及文件 |
|------|------|------|----------|
| ... | ... | ... | ... |

### 🟢 P2 低优先级 (锦上添花)
| 序号 | 任务 | 状态 | 涉及文件 |
|------|------|------|----------|
| ... | ... | ... | ... |

### 🔧 技术债务
| 序号 | 任务 | 状态 |
|------|------|------|
| ... | ... | ... |

---
**统计**: P0 待办 X 项 | P1 待办 X 项 | P2 待办 X 项 | 已完成 X 项
```

## 示例

**用户**: `/read-todo`

**助手**:
```markdown
## 📋 PIA 待办任务

### 🔴 P0 高优先级 (影响核心体验)

1. **P03 精准问答快捷问题按钮** ❌
   - 涉及文件: DetailsPage.tsx, QuickQuestionBox.tsx
   - 说明: 后端 API 已存在，前端未集成

2. **P03 PDF 片段浮层集成** ❌
   - 涉及文件: DetailsPage.tsx, PdfFragmentOverlay.tsx
   - 说明: PdfFragmentOverlay 组件已存在，但未在 DetailsPage 中使用

### 🟡 P1 中优先级 (影响用户效率)

5. **检索结果"仅摘要 vs 有PDF"区分** ❌
   - 涉及文件: ResultsPage.tsx, services/api.ts

...

---
**统计**: P0 待办 4 项 | P1 待办 5 项 | P2 待办 5 项 | 已完成 0 项

请问要开始哪个任务？
```
