## 审计方法论

### 审计维度

| 维度 | 说明 | 优先级 |
|------|------|--------|
| 功能完整性 | PRD 定义的功能是否实现 | P0 |
| API 契约一致性 | 前后端接口是否匹配架构文档 | P0 |
| 数据模型一致性 | 实体定义是否符合 PRD 第 13 章 | P1 |
| 交互规范一致性 | UI/UX 是否符合 PRD 全局规范 | P1 |
| 性能约束符合度 | 是否满足 PRD 第 14 章性能要求 | P2 |

### 审计流程

```
1. 阅读 PRD/架构文档中的模块定义
2. 检查对应代码文件是否存在
3. 对比功能点与实现
4. 检查 API 契约匹配度
5. 记录差距项
```

### 状态标记

- ✅ 完成 - 功能完整实现
- 🟡 部分完成 - 功能部分实现或有缺失
- 🔴 未实现 - 功能完全缺失
- ⚪ 不适用 - 本次审计范围外
