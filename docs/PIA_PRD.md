## 1. 产品概述

SIA 是一款面向科研人员的 **私有文献智能分析助理**。用户通过上传本地 PDF 文献，系统自动利用 AI 进行深度解析、关键指标提取及跨文献综合综述，旨在解决科研人员面对大量已下载文献"读不完、记不住、比不了"的核心痛点。

### 1.1 核心设计理念（V2.1 新增）

**以科研人员任务为中心，而非以系统功能为中心：**

| 传统工具思维 | SIA 产品思维 |
|-------------|-------------|
| 先建项目，再添加文献 | 有文献就先用，项目是事后组织工具 |
| 全量提取所有字段 | 按需精准回答问题，想问什么就提取什么 |
| 检索/上传/DOI 解析分三路 | 统一"添加文献"入口，系统自动识别输入类型 |
| 对比前弹窗预警 | 对比时按条件筛选分组，预警内联到单元格 |
| 导出 CSV/Excel | 一键复制 LaTeX 表格 / 导出图片 |

### 1.2 技术栈约定（影响交互设计）

| 层次 | 选型 | 交互影响 |
|------|------|---------|
| 前端 | React + Tailwind CSS + PDF.js | 支持 PDF 片段浮层预览；响应式布局 |
| 后端 | FastAPI + ThreadPoolExecutor | 异步任务推送；SSE 实时进度流 |
| AI 两阶段 | Stage1/Stage2: OpenAI 兼容接口（用户自配 Base URL + 模型名） | 分步进度反馈；两阶段状态可视化 |
| AI 精准问答 | RAG + LLM | 按需提问提取，响应 <5s，成本 ~$0.005/问 |
| Embedding | BGE-base-en-v1.5 本地内置 | 项目自带，离线可用，无需下载及 API |
| 解析 | Marker / Docling 转 Markdown | 科学公式/表格正常渲染 |
| 缓存 | DOI 级本地加密存储 | 命中缓存时"即时响应"体验 |
| 数据处理 | 化学组分归一化 + 单位自动换算 | 提升检索精度与数据可比性 |
| 外部集成 | OpenAlex, Semantic Scholar | 双引擎检索；支持自然语言意图 |
| AI 接口 | OpenAI 兼容格式（统一 Base URL + Key + 模型名） | 支持 OpenAI/DeepSeek/通义千问/Ollama 等 |
| 领域聚焦 | 钙钛矿 + 半导体优先 | 内置高质量 Schema + 规则校验 |

### 1.3 领域聚焦策略（V2.1 新增）

**先做透，再扩张：**

| 阶段 | 支持领域 | Schema 策略 |
|------|---------|------------|
| V2.1 MVP | 钙钛矿太阳能电池 | 内置完整 Schema + 规则校验 + ISOS 协议识别 |
| V2.2 扩展 | + 半导体/氧化物 | 复用 Schema 框架，新增 Mobility/带隙等指标 |
| V2.3+ | 其他 STEM 领域 | AI 动态生成 Schema（无规则校验兜底，标注"实验性"） |

**边界说明：**
- 非聚焦领域的提取质量可能不稳定，系统会明确标注"该领域提取结果仅供参考"
- 用户可手动校正提取结果，系统学习用户反馈

## 2. 用户角色与核心场景

### 2.1 用户角色

| 角色 | 描述 | 核心诉求 |
|------|------|---------|
| 博士生 (PhD) | 入门到中期，需大量阅读综述与追踪最新进展 | 快速筛选相关文献；工艺参数复现 |
| 博士后 (Postdoc) | 独立研究，需严谨的数据对比与方法论参照 | 迟滞/SPO 标注；ISOS 协议对比 |
| PI（课题组长） | 战略决策，关注领域格局与竞争态势 | 高效综述；导出报告供汇报使用 |

### 2.2 核心使用场景

**场景 A — 单篇快速验证（V2.1 重点优化）**
博士生刚读完一篇好文章，想知道"这篇的退火温度是多少"或"有没有 SPO 数据"。直接拖入 PDF，提问即得答案，3 秒内响应，无需创建项目、无需全量提取。

**场景 B — 批量文献对比**
博士后收集了 20 篇同一方向的论文，想对比"只看 R-scan 的 PCE 数据"。上传后进入对比看板，按条件筛选分组，一键导出 LaTeX 表格用于论文写作。

**场景 C — 跨项目综合问答**
PI 同时管理"钙钛矿稳定性"和"新型钝化材料"两个项目，想问"这两个方向在退火工艺上有什么共性？"。Multi-Doc Chat 支持跨项目选文献进行综合分析。

**场景 D — 无 PDF 快速筛选**
学生输入研究方向关键词，系统检索 Semantic Scholar/OpenAlex 返回摘要列表，Stage1 快速标注相关性。筛选后再下载 PDF 进行深度提取。

## 4. 全局规范

### 4.1 色彩与状态语义

| 状态 | 颜色语义 | 适用场景 |
|------|---------|---------|
| 成功/正常 | 绿色 (#22C55E) | 缓存命中、提取完成、连通正常 |
| 警告 | 橙色 (#F97316) | 数据不可比、迟滞未标注 |
| 错误 | 红色 (#EF4444) | 网络失败、解析出错、权限拒绝 |
| 信息/进行中 | 蓝色 (#3B82F6) | 加载中、AI 处理中、缓存写入 |
| 禁用 | 灰色 (#9CA3AF) | 不可操作状态 |

### 4.2 进度反馈规范（V2.1 重点优化）

**阶段式进度反馈（替代纯百分比）：**

所有耗时操作必须显示**当前阶段**而非仅百分比：

```
┌────────────────────────────────────────────────────────────┐
│  🔬 正在提取...                                            │
│                                                            │
│  ✅ 下载 PDF                              完成 (2s)        │
│  ✅ 解析文档结构                          完成 (3s)        │
│  🔄 AI 正在提取性能指标                    67%             │
│     ████████████████░░░░░░░░                               │
│  ⏳ AI 提取工艺参数                        等待中          │
│  ⏳ AI 提取稳定性数据                      等待中          │
│                                                            │
│  预计剩余时间: 约 20 秒                                    │
└────────────────────────────────────────────────────────────┘
```

**阶段定义：**
| 阶段 | 名称 | 典型耗时 | 可取消 |
|------|------|---------|-------|
| 1 | 下载 PDF | 2-5s | ✅ |
| 2 | 解析文档结构 | 3-5s | ✅ |
| 3 | AI 提取性能指标 | 10-15s | ✅ |
| 4 | AI 提取工艺参数 | 15-20s | ✅ |
| 5 | AI 提取稳定性数据 | 5-10s | ✅ |

### 4.3 空状态规范

所有列表/表格在无数据时必须展示：图标 + 主文案 + 副文案 + 可选操作按钮。

### 5.3 三步引导内容

| 步骤 | 内容 | 必填 | 验证 |
|------|------|------|------|
| 1 | AI 引擎配置（Base URL + API Key + 模型名称） | Base URL + Key 必填（本地部署 Key 可留空）；模型名必填 | 连通性测试（发送测试请求验证 Key + 模型可用性）；Embedding 模型已内置 |
| 2 | 代理配置 | 可选 | 明确提示"不配置则无法下载付费文献 SI" |
| 3 | 研究领域选择 | 必选 | 影响默认 Schema；可多选 |

### 5.4 交互规则

- 弹窗外点击不关闭（强制完成引导）
- API Key 格式验证失败时禁止进入下一步
- 可点击"跳过，稍后配置"但每次启动会重新弹出直到配置完成
- 引导完成后自动跳转至 P01

### 6.3 统一添加文献入口

**三合一输入框，系统自动识别输入类型：**

| 输入类型 | 识别规则 | 后端路径 |
|---------|---------|---------|
| PDF 文件 | 拖拽或点击上传 | 直接解析本地文件 |
| DOI | 匹配 `10.XXXX/XXXX` 格式 | 下载 PDF 后解析 |
| 研究问题/关键词 | 自然语言文本 | 检索 Semantic Scholar/OpenAlex |

**交互流程：**

```
用户输入
    │
    ├── 检测到 PDF 文件 ──→ 直接进入 P01b 快捷模式（单篇处理）
    │
    ├── 检测到 DOI ──→ 弹出确认："检测到 DOI，是否解析该论文？"
    │                   └── 确认 → 下载 PDF → 进入 P01b
    │
    └── 检测到文本 ──→ 执行检索 → 返回摘要列表
                        └── 点击某篇 → 可选择"下载 PDF 深度提取"或"仅查看摘要"
```

### 6.4 临时收集箱

**未归档文献的默认存放位置：**

- 所有通过快捷入口添加的文献默认进入临时收集箱
- 用户可随时将文献"收纳到项目"或删除
- 临时收集箱的文献同样支持问答、提取、对比
- 体现"项目是事后组织工具"的设计理念

## 11. 全局异常处理规范

### 11.1 网络层异常

| HTTP 状态码 | 含义 | 全局处理策略 |
|------------|------|------------|
| 401 | 认证失效 | Toast 提示"会话已过期，请重新配置 Cookie" + [前往配置] |
| 403 | 权限不足（付费墙） | Toast "该文献需机构订阅权限" + [配置代理] |
| 404 | 资源不存在 | 内联提示"未找到对应资源，DOI 可能有误" |
| 429 | API 限流 | Toast "AI 接口调用频率超限，15s 后自动重试" |
| 500/502 | 服务器错误 | Toast "服务暂时不可用，请稍后重试" + 错误 ID |

### 11.2 AI 提取质量异常

| 质量问题 | 检测规则 | 处理策略 |
|---------|---------|---------|
| 字段完整度低 | 提取字段 < 设定阈值 | 页面顶部黄色横幅 + 提示缺失字段数量 |
| 数值超出合理范围 | PCE > 32% 或 < 1% | 字段标红 + "数值可能有误，建议人工核查" |
| 单位不一致 | 混用 mA vs A | 自动换算 + Toast"已自动统一单位" |

### 11.3 系统级降级策略

- **Stage2 不可用**：自动降级到 Stage1 模型 + 精准问答模式
- **所有 AI 服务不可用**：允许用户手动输入/编辑参数；页面进入"离线模式"
- **非核心领域**：标注"实验性支持，提取结果仅供参考"

## 12. 核心算法与数据策略

### 12.1 化学组分归一化

*   **痛点**：钙钛矿组分描述极其混乱
*   **策略**：正则解析 + 元素映射 + 语义向量化

### 12.2 自动单位换算

*   **策略**：基于正则识别 + `pint` 库自动换算

### 12.3 阶段式进度反馈（V2.1 新增）

```typescript
interface ExtractionProgress {
  stages: [
    { name: "下载 PDF", status: "completed", duration: 2 },
    { name: "解析文档结构", status: "completed", duration: 3 },
    { name: "AI 提取性能指标", status: "in_progress", progress: 67 },
    { name: "AI 提取工艺参数", status: "pending" },
    { name: "AI 提取稳定性数据", status: "pending" }
  ];
  estimatedRemaining: 20; // seconds
}
```

### 12.4 精准问答引擎（V2.1 新增）

**基于 RAG 的按需提取：**

```
用户问题 → 向量检索相关段落 → LLM 精准回答 → 返回答案 + 来源定位
```

- **向量引擎**：FAISS (本地运行)
- **Embedding 模型**：`BAAI/bge-base-en-v1.5` 本地内置（~420MB，首次启动自动下载，MIT 协议，768 维，科研文献检索 MTEB 得分优异）
- **Chunk 策略**：按段落切片，512 tokens，重叠 64 tokens
- **响应速度**：<5s
- **成本**：~$0.005/问
