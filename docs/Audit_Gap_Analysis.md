# SIA V2.1 审计报告与修复方案 (2026-05-11)

## 1. 审计综述 (Executive Summary)

经过对 `docs/PIA_PRD.md`、`docs/Refactor_Plan.md` 以及 `src/`、`src-python/` 目录下代码的对比分析，结论如下：

*   **后端 (Backend)**: 已完成 80% 的模块化重构。6 实体数据库模型、Stage1/Stage2 提取引擎、RAG 问答引擎、配置加密系统已就绪。
*   **前端 (Frontend)**: 处于“新瓶装旧酒”状态。虽然路由和页面结构已建立，但 P01 首页、P04 对比页的核心逻辑仍停留在 V1 阶段，缺失关键业务组件。
*   **整体状态**: 处于重构 Phase 1 与 Phase 2 之间，前端落后于后端进度。

---

## 2. 发现的不一致项 (Identified Gaps)

| 模块 | 需求 (PRD/Plan) | 现状 (Code) | 结论 |
| :--- | :--- | :--- | :--- |
| **首页 P01** | 三合一输入 + 临时收集箱 + 项目门户 | 仅有搜索框 + 简单历史记录 | 🔴 **严重不一致** |
| **项目枢纽 P02** | 分析矩阵 + 多文档问答 (Multi-Doc Chat) | 仅有文献列表，无 Q&A 侧边栏 | 🟠 **功能缺失** |
| **对比看板 P04** | 条件筛选 + 内联预警 + LaTeX/PNG 导出 | 静态 V1 表格，仅支持 Excel 导出 | 🟠 **功能缺失** |
| **统一输入框** | 自动识别 DOI/PDF/Keyword | `UnifiedInputBox` 组件不存在 | 🔴 **核心缺失** |
| **后端入口** | 极致瘦身的 `main.py` (~50行) | 530行混合代码，存在 V1/V2 逻辑冗余 | 🟡 **技术债** |
| **命名规范** | `ComparePage.tsx` | 实际为 `ComparisonPage.tsx` | ⚪ **琐碎项** |

## 4. 思考过程 (Thought Process)

1.  **第一性原理分析**: SIA 的核心价值是“以任务为中心”。目前的 HomePage 强制用户进行“搜索”，而非“管理文献”，这违背了 PRD 中“项目是事后组织工具”的理念。必须优先重写首页。
2.  **风险评估**: 后端虽然重构了，但 `main.py` 中的冗余代码可能导致前端调用了旧的 V1 接口，产生数据不一致。需要确保前端全面转向 `services/projectApi.ts` 等新服务。
3.  **一致性修正**: 将 `ComparisonPage.tsx` 重命名为 `ComparePage.tsx` 以对齐文档。
